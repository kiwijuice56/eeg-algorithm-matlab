# EEG/fNIRS MATLAB Algorithms
A repository that implements basic EEG and fNIRS data analysis for use
in a cognitive impairment screening mobile app.

## Study schema
The app records data using the Muse S Athena headband on preset 1042 (high power fNIRS and full EEG).

Four tasks are use to stimulate brain data. 

| Task JSON Key | Task Description | Length | Notes |
| --- | --- | --- | --- |
| `no_stimulus` | No stimulus task (resting state) | 60 seconds | |
| `assr_listening` | 40 Hz and 45 Hz audio listening task, high and low volume variants for each (steady state response) | 60 seconds for each audio | Because this task has changed over time, some old recordings compare different frequencies such as 80 Hz or 20 Hz |
| `oddball` | Oddball audio listening task (event related potentials) | 94 seconds (25 rare tones, 100 common tones) | |
| `reaction` | Reaction time task (energetics) | ~60 seconds (20 measurements every 1.5 - 3.0 seconds) | |

## Code base
| Directory | Purpose | 
| --- | --- |
| `test/` | Small independent scripts to visualize data, usually for paper figures |
| `helper/` | Functions used across the code-base to accomplish general tasks |
| `features/` | Functions and scripts used to extract more advanced features from data |
| `app_functions/` | Code that will be translated to C++ and included in the final app |

## Data format
The data is stored in a JSON format with a key : dictionary entry for each of the four tasks. 
Each task dictionary holds a key : dictionary entry for all data streams collected by the Muse headband (e.g. `eeg`, `optics` for fNIRS).
The dictionary for each data stream contains two keys: `time` and `data`; 
The `time` entry maps to an array of integer timestamps while the `data` entry maps to
an array of arrays, one for each data point at the corresponding timestamp. 
The size of each data point array depends on the number of channels in that particular data stream.
For example, EEG holds eight values while fNIRS holds sixteen values (see the description for each index below).


### EEG
#### Summary (from Muse documentation)
Raw EEG values are given in microvolts. 

#### Packet indices
| Index (1-indexed) | Description |
|---|---|

| 8 | Auxiliary input 4 on MuseS 2025 (unused) |


### fNIRS (optics)
#### Summary (from Muse documentation)
Functional near-infrared spectroscopy (fNIRS) is a non-invasive imaging technique that measures brain activity by detecting changes in blood oxygenation.
The light absorbtion of the blood varies by wavelength. 
The 750nm wavelength is sensitive to deoxygenated hemoglobin (HbR), while the 850nm wavelength is sensitive to oxygenated hemoglobin (HbO). 
By analyzing the absorption of these wavelengths, we can determine changes in blood oxygenation levels.
The Optics data is also used to measure PPG.
Optics data values are given in microamps (uA).

### Metadata
Each task also holds extra information that is generated by the app itself.
| JSON Key | Description |
|---|---|
| `stimulus_unix_time` | Exact unix timestamps (floating point, seconds) at which stimuli were presented |
| `stimulus_label` | String labels of the stimuli presentations, same length as `stimulus_time`. Primarily used for the oddball task in which each stimuli can be either `common` or `rare` |
| `response_unix_time` | Exact unix timestamps at which the user inputted a response (used only for the reaction time task) |
| `response_label` | String labels of user responses; same length as `response_time` |
| `task_start_unix_time` | Exact unix timestamp at which the task began |
| `task_end_unix_time` | Exact unix timestamp at which the task ended |
